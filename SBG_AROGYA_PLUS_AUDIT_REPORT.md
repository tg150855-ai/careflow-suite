# SBG Arogya Plus HMIS Audit Report

## Scope completed in this pass

- Focused audit and repair of the Patient Registration and Patient Management module.
- Checked sidebar route registration for all 88 sidebar destinations.
- Checked patient database schema, row-level access rules, table grants, relationships, and indexes.
- Checked dashboard/command-center queries for broken table and column references found during the audit.

## Working now

- Sidebar navigation: all sidebar links resolve to existing route files.
- Patients list route: `/patients` loads from the real `patients` table.
- New Patient button: opens `/patients/new`.
- Register First Patient button: opens `/patients/new`.
- Patient registration form: saves real data to Supabase/Lovable Cloud.
- UHID: auto-generated by the database using the existing `generate_uhid()` function.
- Patient search: works by name, mobile, and UHID.
- Patient filtering: gender filter added.
- Patient pagination: page controls added.
- Patient record view: opens `/patients/:id` and loads the unified patient workspace.
- Patient edit: available from the patient workspace modal and list edit action.
- Patient delete: enabled for roles that have delete permission, with confirmation and audit logging.
- Patient insurance: registration/edit form can save linked policy details to `patient_insurance`.
- Global search: includes patients, bills, medicines, and lab orders.
- Audit logging: patient create/update/delete/view/print actions call the existing audit logger.

## Broken items found and fixed

- `patients`, `patient_insurance`, and `insurance_companies` had no effective Data API grants for app roles, blocking real reads/writes from the UI.
- `patient_insurance.patient_id` had no foreign key to `patients.id`; now linked with cascade behavior.
- Patient search needed proper indexes; added UHID, mobile, name trigram, and insurance lookup indexes.
- Dashboard and command center had stale schema references:
  - `emergency_visits` replaced with `emergency_cases`.
  - `bills.total_amount` replaced with `bills.total`.
  - `bills.balance_amount` replaced with `bills.pending`.
  - `admissions.discharge_date` replaced with `admissions.discharged_at`.
  - `admissions.admission_date` replaced with `admissions.admitted_at`.
  - `surgeries.scheduled_at` replaced with `surgeries.scheduled_start`.
  - `pharmacy_sales.total_amount` replaced with `pharmacy_sales.total`.
  - `medicines.current_stock` replaced with `medicine_batches.quantity` for low-stock checks.
  - `command-center` revenue now uses `bills.total` instead of a missing `net_amount`.
